# 計画の実行

## 概要

計画を読み込み、批判的にレビューし、タスクをバッチで実行し、バッチ間でレビュー用に報告します。

**中核原則:** アーキテクトレビューのためのチェックポイント付きバッチ実行。

**開始時に宣言:** 「executing-plans の skill を使ってこの計画を実装します。」

## プロセス

### ステップ 1: 計画の読み込みとレビュー
1. 計画ファイルを読む
2. 批判的にレビューし、疑問点や懸念を特定する
3. 懸念がある場合: 開始前に人間パートナーへ共有する
4. 懸念がない場合: TodoWrite を作成して進む

### ステップ 2: バッチ実行
**デフォルト: 最初の 3 タスク**

各タスクについて:
1. in_progress にする
2. 各ステップを正確に実行する (計画は小さなステップで構成)
3. 指定された検証を実行する
4. completed にする

### ステップ 3: 報告
バッチ完了時:
- 実装した内容を示す
- 検証出力を示す
- 「Ready for feedback.」と言う

### ステップ 4: 続行
フィードバックに基づいて:
- 必要なら変更を適用する
- 次のバッチを実行する
- 完了まで繰り返す

### ステップ 5: 開発の完了

すべてのタスクが完了し検証されたら:
- 「finishing-a-development-branch の skill を使ってこの作業を完了します」と宣言する
- **必須サブスキル:** superpowers:finishing-a-development-branch を使用
- その skill に従い、テスト確認、選択肢提示、選択の実行を行う

## 停止して助けを求めるタイミング

**次の場合は直ちに実行を停止:**
- バッチ途中でブロッカーに遭遇 (依存不足、テスト失敗、指示が不明)
- 計画に重大な欠落があり開始できない
- 指示が理解できない
- 検証が繰り返し失敗する

**推測せず、確認を求めること。**

## 以前のステップに戻るタイミング

**レビュー (ステップ 1) に戻るのは:**
- パートナーがあなたのフィードバックに基づいて計画を更新したとき
- 根本的なアプローチの見直しが必要なとき

**ブロッカーを無理に突破しない** - 停止して確認する。

## 覚えておくこと
- まず計画を批判的にレビューする
- 計画のステップを正確に守る
- 検証を省略しない
- 計画で指定された skills を参照する
- バッチ間は報告して待つだけ
- ブロックされたら止まり、推測しない
